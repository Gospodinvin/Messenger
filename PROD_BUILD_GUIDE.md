# Руководство по компиляции и подготовке продакшен-версии APK установочника 📱💼
## Проект: TeamMate Pro (React Native + Expo SDK 52)

Это подробное руководство описывает процесс сборки, цифровой подписи и оптимизации автономного файла **.APK** для последующей дистрибуции и продажи конечным пользователям.

---

## 🛠️ Введение: Архитектура дистрибуции
Поскольку данное веб-окружение является изолированной интерактивной спецификацией и симулятором работы приложения, физический запуск тяжелого Gradle компилятора и Android NDK недоступен внутри данного контейнера (из-за отсутствия локальных SDK-инструментов, ключей подписи Keystore и ограничений оперативной памяти). 

Сборка мобильного APK осуществляется через облачную систему **Expo EAS (Expo Application Services)** или локально на вашем компьютере. Мы добавили полные преднастроенные файлы конфигурации:
* **`app.json`** — параметры пакета Android (`com.teammate.pro`), привилегии камеры, микрофона, биометрии Face ID/Touch ID.
* **`eas.json`** — профили сборки, включая профиль `preview` для мгновенной генерации прямого дистрибутивного **.APK** файла (в обход Google Play) и профиль `production` для итогового **.AAB** bundle-файла.

## 🚀 Пошаговый алгоритм компиляции продакшен APK

### Шаг 1: Авторизация в облачной системе сборки Expo
Сборка через EAS полностью бесплатна и не требует наличия мощного компьютера или macOS. В терминале на вашем ПК выполните:
```bash
# Установите CLI утилиту EAS, если она ещё не установлена
npm install -g eas-cli

# Авторизуйтесь под своей учетной записью Expo (создается бесплатно на expo.dev)
eas login
```

### Шаг 2: Связывание проекта с аккаунтом Expo
Выполните инициализацию проекта в вашем облачном пространстве разработчика:
```bash
eas project:init
```

### Шаг 3: Запуск сборки дистрибутивного APK (Релизная сборка)
Для прямой продажи пользователям (в обход Google Play Store) нужен самодостаточный установочный файл **.APK**, который пользователи скачивают и устанавливают в один клик. Профиль `preview` в нашем `eas.json` настроен именно так:
```bash
eas build --platform android --profile preview
```

#### Что произойдет автоматически на серверах Expo:
1. **Пребилд**: Сгенерируется чистый нативный проект Android со всеми зависимостями (WebRTC, SQLCipher, Биометрия, Mammoth.js и т.д.).
2. **Генерация ключа подписи (Keystore)**: EAS автоматически создаст уникальное цифровое хранилище ключей Android Keystore для вашего приложения и сохранит его в вашем защищенном аккаунте Expo.
3. **Компиляция**: На удаленном сервере запустится сборка `Release.apk`.
4. **Оптимизация**: Код сожмется с помощью R8/ProGuard, повышая скорость выполнения и защищая исходный код от декомпиляции.
5. **Выпуск**: По завершении сборки (обычно 5–10 минут) вам выдадут прямую ссылку на скачивание готового файла `teammate-pro.apk` и QR-код для установки прямо на телефон!

## 🎯 Подготовка к продаже конечным пользователям

При распространении закрытого корпоративного софта для продажи напрямую (без маркетплейсов), соблюдайте следующие правила безопасности и коммерческой упаковки:

### 1. Безопасность ключей подписи (Keystore)
Никому не передавайте и обязательно сохраните локально файл вашего Android KeyStore. Если вы в будущем захотите выпустить обновление, вам потребуется использовать **тот же самый ключ**:
```bash
# Команда для получения/бэкапа вашего сгенерированного keystore-файла
eas credentials
```

### 2. Организация защиты и Лицензирования
Поскольку TeamMate Pro хранит всю переписку, задачи и документы локально внутри шифрованного контейнера **SQLCipher AES-256**, данные пользователей в безопасности. Для продажи ПО вы можете использовать:
* **Провайдер авторизации (Supabase/Firebase)**: Регистрация пользователей по закрытым приглашениям (инвайт-кодам).
* **Локальный Лицензионный Ключ**: При первом запуске вшивать проверку хэша лицензии, генерируемого на основе UUID устройства (через `expo-application` API).

### 3. Гайд по установке для ваших покупателей
Так как приложение устанавливается напрямую из APK (Sideloading), подготовьте для покупателей следующую инструкцию:
1. Скачать файл `teammate-pro.apk` на смартфон.
2. Нажать на файл и разрешить установку приложений из "Неизвестных источников" (политика безопасности Android).
3. При первом входе задать 4-значный мастер-пароль приложения (используйте тестовый код `1234` для демонстрации).
4. Разрешить доступ к системной биометрии (Face ID / Отпечаток пальца) для последующего беспарольного быстрого входа.

---

### 🛡️ Предотвращение ложных срабатываний антивирусов (Google Play Protect)
При прямой дистрибуции релизных APK антивирусные системы могут иногда предупреждать о "Неизвестном издателе". Для минимизации этого:
* Подпишите приложение легитимным SSL-сертификатом компании разработчика.
* Или отправьте собранный APK на валидацию в форму [Google Play Protect Appeal](https://support.google.com/googleplay/android-developer/contact/protect_appeals) — они добавят хэш-сумму вашего APK в доверенную глобальную базу в течение 2-3 рабочих дней, и у покупателей больше не возникнет всплывающих предупреждений при установке.
